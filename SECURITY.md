# Security Policy

We take the security of our website, data, and users very seriously. If you look at or interact with our site and discover any security vulnerabilities, we appreciate your help in disclosing them to us in a responsible manner.

## Supported Versions

Only the latest version of the website currently deployed on the `main` branch is actively supported with security updates. 

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| Older   | :x:                |

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues or public comments. Instead, report them privately to ensure the safety of our users.

To report a vulnerability, please follow these steps:
1. Send an email to **smithbush42@gmail.com** with the subject line `Vulnerability Report`.
2. Include a detailed description of the vulnerability, including the specific page or component affected.
3. Provide step-by-step instructions or a proof-of-concept to help us reproduce the issue.
4. Let us know if you would like to be credited publicly once the issue is resolved.

## Our Commitment

When you report a vulnerability responsibly, we commit to:
* Acknowledging receipt of your report within **48 hours**.
* Providing a targeted timeline for fixing the issue.
* Keeping you informed of our progress as we work on a resolution.
* Not taking legal action against you if you follow this policy and do not exploit the vulnerability maliciously.

Thank you for helping keep our platform safe!
